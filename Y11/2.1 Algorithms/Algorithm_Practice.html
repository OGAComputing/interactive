Use a sentinel-controlled loop to log training run paces, then output a full statistical summary.

Scenario

A runner uses an app to log their pace (in whole minutes per km) after each training run. The app keeps asking for a pace until the runner enters −1 as a sentinel value to stop. It then outputs: total runs logged, average pace, fastest (lowest) pace, and slowest (highest) pace.

If the runner enters −1 immediately (no runs logged), the app outputs No runs recorded instead.

Your algorithm must:

  • Initialise variables for the run count, total pace, fastest pace, and slowest pace
  • Use a condition-controlled loop (while or do…until) that stops when −1 is entered
  • On each valid iteration: add the pace to the total, increment the count, update fastest and slowest
  • Not include the sentinel value (−1) in any calculations
  • If count > 0: output run count, average pace, fastest, and slowest
  • If count = 0: output No runs recorded
Test case hint: With paces 12, 20, 16 then −1 → count=3, average=16, fastest=12, slowest=20. Your output should contain these values so the checker can verify each criterion.
