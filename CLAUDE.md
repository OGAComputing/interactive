# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Interactive Computing Education Platform — a static web hub serving self-contained HTML-based CS activities for UK secondary school students (Years 8, 9, 11). Zero dependencies, no build step, no package manager.

## Architecture

- **`index.html`** — Hub page that fetches `activities.json` and `meta.json` to dynamically render activity cards grouped by year and topic
- **`Y8/`, `Y9/`, `Y11/`, `Demo/`** — Year group folders containing topic subfolders with standalone activity HTML files
- **`activities.json`** and **`meta.json`** — Auto-generated by GitHub Actions workflow on push to main; never edit manually
- **`.github/workflows/build-manifest.yml`** — Parses `<!-- ACTIVITY -->` comment blocks from HTML files to rebuild the manifest files

## Running Locally

Open `index.html` in a browser. No server or build step required. Each activity HTML file can also be opened directly.

## Adding a New Activity

1. Create an HTML file under the appropriate folder: `Y<number>/<Topic>/filename.html`
2. Include the required metadata comment block at the top:
   ```html
   <!-- ACTIVITY
   title:  Activity Title
   type:   Practice or Challenge
   stars:  1-4
   desc:   Short description for the hub card
   color:  teal|amber|red|green|blue|cyan|purple|indigo
   -->
   ```
3. Implement the activity as a fully self-contained HTML page (inline CSS and JS, no external dependencies)
4. Push to main — the workflow auto-generates `activities.json` and `meta.json`

## Key Conventions

- **Self-contained HTML**: Every activity must work as a standalone file with inline styles and scripts — no external CSS/JS
- **Vanilla JS only**: No frameworks or libraries; direct DOM manipulation
- **Files/folders starting with `_`** are excluded from the manifest (use `_drafts/` for WIP)
- **Folder structure determines metadata**: `Y8/` → "Year 8", folder names become topic labels
- **Color themes**: Activities specify a color from the fixed palette that controls card border and badge colors in the hub
